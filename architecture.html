什么是 Function Calling？（30 秒理解核心概念）

目前你的 Agent 完全依赖 LLM 的"记忆"来回答单词问题。而 Function Calling 让 LLM 拥有了"手"——

当 LLM 觉得自己需要查字典时，它不会直接编造答案，而是返回一个结构化的函数调用请求（比如 {"name": "lookupWord", "arguments": {"word": "resilient"}}）。

你的代码捕获这个请求，真正调用外部字典 API 获取准确数据，再把结果喂回给 LLM。

LLM 拿到真实数据后，再用它的语言能力组织成一个优雅的教学回复。

简而言之：LLM 负责"思考和表达"，Function 负责"获取事实"。两者各司其职。

调用 LLM + Tool Use Loop （新增）
核心循环逻辑：

while (true) {
  ① 调用 client.chat.completions.create({ messages, tools })
  ② 检查返回的 message.tool_calls
  ③ 如果有 tool_calls：
    — 把 assistant message（含 tool_calls）追加到 messages
    — 遍历每个 tool_call，执行对应函数（如 lookupWord("resilient")）
    — 把函数返回值作为 role: "tool" 消息追加到 messages
    — 继续循环（让 LLM 基于工具返回的数据生成回复）
  ④ 如果没有 tool_calls（有 content）：
    — 跳出循环，返回 message.content 作为最终回复
}
更新对话历史 （现有，不改动）

五、一次完整调用的数据流（以查询 "resilient" 为例）
# 谁 → 谁 数据内容
1 用户 → Agent "resilient 是什么意思？"
2 Agent → LLM classify() 调用 → 返回 VOCABULARY
3 Agent → LLM 发送 messages（含 system+cot+fewshot+history+user）+ tools: [dictionaryTool]
4 LLM → Agent 返回 tool_calls: [{ function: { name: "lookupWord", arguments: '{"word":"resilient"}' } }]
5 Agent → API HTTP GET https://api.dictionaryapi.dev/api/v2/entries/en/resilient
6 API → Agent 返回 JSON：音标 /rɪˈzɪliənt/、释义 "able to recover quickly"、例句、同义词等
7 Agent → LLM 把 API 数据作为 role: "tool" 消息追加，再次调用 LLM
8 LLM → Agent 基于真实字典数据，生成结构化的教学回复（含音标、释义、例句、Pro tip）
9 Agent → 用户 输出最终教学回复
